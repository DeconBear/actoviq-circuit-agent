# Specification Schema (spec.normalized.json)

The normalized spec is the primary contract for the design workflow. Generated by `scripts/normalize_spec.py`.

## Required Fields

| Field | Type | Description |
|---|---|---|
| `project_name` | string | Snake_case project name |
| `domain` | string | Circuit domain: `opamp`, `oscillator`, `lna_rf`, `power`, `filter`, `comparator`, `ldo`, `mixer` |
| `input_node` | string | Net name of the input signal node |
| `output_node` | string | Net name of the output signal node |
| `targets` | object | Key-value pairs of target name → specification rules |

## Target Specification Rules

```json
{
  "target_name": { "min": 10.0 },
  "target_name": { "max": 5.0 },
  "target_name": { "value": 3.3 },
  "target_name": { "min": 1000.0, "max": 2000.0 }
}
```

## Optional Fields

| Field | Type | Description |
|---|---|---|
| `constraints` | object | Design constraints: `topology`, `supply_v`, `output_logic`, `suggest_partitioned_design` |
| `notes` | string | Additional design notes |
| `targets_eval` | object | Evaluation targets (generated by normalize_spec) |
| `metric_aliases` | object | Alias mappings (generated by normalize_spec) |

## Example

```json
{
  "project_name": "rc_lowpass_filter",
  "domain": "filter",
  "input_node": "vin",
  "output_node": "vout",
  "supply_node": "vcc",
  "constraints": {
    "topology": "rc_lowpass",
    "suggest_partitioned_design": false
  },
  "targets": {
    "cutoff_hz": { "min": 900, "max": 1100 },
    "gain_db_100hz": { "min": -0.5 }
  }
}
```
